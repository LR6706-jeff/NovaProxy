# NovaProxy 公众号写作 Brief

## 核心需求
通过通俗易懂、具有感染力的文字，教大家如何联动 NovaProxy 和 cc-switch，在 Claude Code 环境下零门槛使用 GLM-4 和 Minimax。

## 目标受众
- 想要使用 Claude Code 提升效率，但受限于 API 访问或费用的国内开发者。
- 喜欢尝试各种模型（如 GLM, Minimax）在实际编程场景中表现的技术爱好者。
- 对终端美学、极简配置有追求的程序员。

## 关键信息
- **NovaProxy**：核心“桥梁”。协议转换 + 高颜值 UI 配置 + 多 Key 轮询。
- **cc-switch**：核心“切换器”。解决 Claude Code 锁定官方 URL 的痛点。
- **组合拳**： cc-switch 指向 NovaProxy -> NovaProxy 桥接 NVIDIA NIM -> 调用国产模型。

## 风格要求
- 口语化、现场感、接地气。
- 拒绝 AI 套话，加入作者的真实吐槽和爽点。
- 强调「实践 + 落地」。

## 字数要求
- 1500 - 2500 字（包含代码块和步骤说明）。
