## 零：准备“能量块”

这一步最简单。我们要薅的是 NVIDIA 的羊毛。

去 NVIDIA NIM 平台寻找 **GLM-4.7** 或者 **Minimax-M2.1**。目前还是慷慨的免费/极低门槛阶段，速度快得惊人。拿到那个 `nvapi-xxx` 格式的密钥（Key）就算成功，把它记声明在小本本上。

---

## 一：架起“跨海大桥”——NovaProxy

由于 Claude Code 只认它自家的协议，所以我们需要一个中间件来做翻译。

这时候就轮到 **NovaProxy** 出场了。

这软件最让我惊喜的是它的审美。双击运行 `NovaProxy.exe`，它会自动弹出个网页后台。

**注意看这个“暖沙色”的界面。** 真的，在被那些冷冰冰、硬邦邦的黑色终端包围久了之后，看到这种温柔的配色简直是种治愈。操作也很绝：

1. 找左边的设置框，把刚才的 NVIDIA Key 直接拍进去。
2. 点击保存。

**完事儿。** 你的本地现在已经跑起了一个“翻译器”，它的接头地址就是：`http://localhost:3001/v1`。

---

## 二：暴力破拆——cc-switch 登场

Claude Code 官方很顽固，它默认只往自家服务器发请求。我们要想让它“改道”，得请出 **cc-switch** 这个神器。

很多人觉得配这种环境变量麻烦。**cc-switch** 的爽点就在于它可以一键切换配置：

1. 安装完之后，在终端加个节点：
   `cc-switch add Nova-GLM`
2. 按提示填地址：`http://localhost:3001/v1`。
3. 执行：`cc-switch switch Nova-GLM`。

这一套操作下来，你就相当于直接把 Claude Code 的“方向盘”给抢过来了。

---

## 三：见证奇迹的时刻

现在，回到你的代码工程目录，深吸一口气，输入：

```bash

claude --dangerously-skip-permissions
```

**重点来了！** 你会发现 Claude 并没有报错，而是顺着我们搭好的“ NovaProxy 桥梁”，直接连上了 NVIDIA 上的 **GLM-4.7** 或者 **Minimax-M2.1**。
